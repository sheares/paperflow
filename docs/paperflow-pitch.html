Why this isn't already solved

The two obvious answers
are each half a product.

"Redact before the LLM"

A commodity.

Reversible redaction with consistent placeholders is already productised by Gravitee, Philter, TrueFoundry and LangChain, and free as MIT open source. But it is built for chat and agent prompts, not messy document piles.

"Cloud IDP reconciliation"

Runs in the wrong place.

Rossum, Infrrd and Super.AI already match fields across documents. But the matching runs in the vendor's cloud, which is exactly what a privacy-blocked organisation cannot use.

Neither pillar alone is the wedge. The intersection is the product.

The wedge

Data minimisation, not data residency.

Data residency · what everyone offers

Bedrock, Vertex, Claude Desktop on third-party. "Bring the model to your cloud." You control where the endpoint runs.

documents→ Farid Hassan→ cloud model

The real identity still reaches the endpoint.

Data minimisation · paperflow

The real identity never reaches the reasoning model at all. Presidio tokenises before anything crosses.

documents→ local map→ [PERSON_1]→ cloud model

Only tokens cross. The map stays on your machine.

Relocating the model is commodity BYO-cloud. Removing the identity is the whole product.

The product

Trust is the interface,
not a footnote.

Per-exchange privacy receipt

Shown inline every turn, never behind a toggle. A routing chip states the route: "Local only · 0 cloud calls" or "Local + Cloud · 1 redacted call".

One entity, one colour, three surfaces

Underline on the real value in your message, a dashed token in the receipt, a solid chip in the record. Hover one and its matches light up across panes.

Pre-send review gate

If a message would cross to the cloud with detected values, the send button arms to "Confirm · send redacted". Human consent before anything leaves.

Detected-entities panel

Every entity Presidio caught, with a "flag a missed one" control. Detection recall you can audit live, not a claim you have to trust.

What we don't claim

Sharply reduced exposure,
not zero leakage.

Security rests on detection recall. Structured identifiers are high-recall; person and company names rely on NER and will miss edge cases. So the UI says "0 detected identifiers crossed", never "zero leakage". The pre-send gate and the detected-entities panel are that caveat rendered as features.

For un-patternable free text, full-local mode is the honest answer, and there the absolute claim is true by construction. We frame paperflow for environments where cloud document AI is off-limits, not as a certified compliance product.
